## 一、現況分析

### 1.1 部署環境
- **前端網站**: www.neovega.cc- 內網: `unified-commerce-hub.zeabur.internal:18789`
  - 外網: `neovegawww.zeabur.app:18789`
- **Openclaw Agent**: 
  - 內網: `openclaw.zeabur.internal:18789`
- **Telegram 聊天室**: Chat ID `-1003806455231`

### 1.2 現有架構

#### 前端聊天元件
- **ChatWidget.tsx**: 商城頁面的浮動聊天視窗
  - 目前使用 mock 回覆
  - 需要整合真實 API

#### 後端服務層
- **chat.ts**: 聊天服務
  - 儲存訊息到 PocketBase
  - `sendChatMessage()` 目前回傳空字串
  
- **api.ts**: API 整合層
  - `callOpenClaw()`: 已實作，可呼叫 Openclaw Agent
  - `callN8nChat()`: n8n webhook 整合
  - `callLLM()`: 通用 LLM API

- **useChat.ts**: React Hook
  - 已整合 OpenClaw、LLM、n8n
  - 支援 fallback 機制
  - **問題**: ChatWidget 未使用此 hook

## 二、整合需求

### 2.1 功能需求
1. 網站訪客在 ChatWidget 發送訊息
2. 訊息轉發到 Openclaw Agent
3. Openclaw 將訊息轉發到 Telegram 聊天室
4. Telegram 中的 agents 回覆
5. 回覆透過 Openclaw 返回網站
6. 訪客在 ChatWidget 看到回覆

### 2.2 技術需求
1. ChatWidget 整合 useChat hook
2. 配置 Openclaw Agent URL
3. Openclaw 需要支援 Telegram 雙向通訊
4. 訊息需要關聯到正確的 conversation

## 四、技術細節

### 4.1 訊息流程圖
```
[網站訪客] ↓ (輸入訊息)
[ChatWidget]
    ↓ (useChat.sendMessage)
[chat.ts: sendChatMessage]
    ↓ (儲存到 PocketBase)
    ↓ (callOpenClaw)
[Openclaw Agent]
    ↓ (轉發)
[Telegram Chat -1003806455231]
    ↓ (agents 回覆)
[Openclaw Agent]
    ↓ (返回)
[chat.ts]
    ↓ (儲存 assistant 訊息)
    ↓ (返回回覆)

    ↓ (顯示)

```

## 五、風險與注意事項

### 5.1 技術風險
1. **Openclaw Telegram 整合未完成**
   - 風險: 需要額外開發時間
   - 緩解: 先確認 Openclaw 現有功能

2. **非同步回覆處理**
   - 風險: Telegram agents 回覆可能需要時間
   - 緩解: 實作 timeout 機制，顯示「處理中」狀態

3. **訊息關聯**
   - 風險: 多個訪客同時聊天，訊息可能混淆
   - 緩解: 使用 userId + conversationId 唯一識別

### 5.2 效能考量
1. **回覆延遲**: Telegram agents 回覆時間不確定
2. **連線穩定性**: Zeabur 內網連線需要穩定
3. **並發處理**: 多訪客同時聊天的負載
